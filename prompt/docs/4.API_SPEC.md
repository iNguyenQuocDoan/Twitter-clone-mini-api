# API Specification

## Overview

| Setting | Value |
|---------|-------|
| Base URL | `/api/v1` |
| Versioning | URL path — breaking changes introduce `/api/v2/` |
| Content-Type | `application/json` (all endpoints except file upload) |
| File upload | `multipart/form-data` |
| Protocol | HTTPS only in production |
| Encoding | UTF-8 |
| Docs | `GET /docs` — Swagger UI (disabled in production) |

## Authentication

### Token Flow

```
POST /api/v1/auth/login
  → Response body: { accessToken }
  → Set-Cookie: refreshToken=<value>; HttpOnly; Secure; SameSite=Strict; Path=/api/v1/auth/refresh
```

| Token | Lifetime | Storage | Delivery |
|-------|----------|---------|----------|
| Access Token | 15 minutes | Client memory (not localStorage) | Authorization header |
| Refresh Token | 7 days | httpOnly cookie | Sent automatically by browser |

### Request Header Format

```
Authorization: Bearer <access_token>
```

### Token Refresh Flow

```
POST /api/v1/auth/refresh
  → Browser sends refreshToken cookie automatically
  → Server: validate hash, check is_revoked, check expires_at
  → Issue new access token + rotate refresh token (invalidate old, issue new)
  → Response: { accessToken } + new Set-Cookie
```

> **Rotation:** Every call to `/auth/refresh` invalidates the current refresh token and issues a new one. If a stolen refresh token is used after the legitimate user has already refreshed, the server detects reuse of an invalidated token and revokes all tokens for that user.

### Auth Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `AUTH_001` | 401 | Invalid email or password |
| `AUTH_002` | 401 | Access token expired |
| `AUTH_003` | 401 | Access token invalid or malformed |
| `AUTH_004` | 401 | Refresh token expired or revoked |
| `AUTH_005` | 403 | Insufficient permissions (wrong role) |
| `AUTH_006` | 409 | Email already registered |

### Route Access Matrix

| Access Level | Mechanism | Who |
|-------------|-----------|-----|
| Public | No header required | Anyone |
| Protected | `Authorization: Bearer <token>` | Any authenticated user |
| Admin | Protected + `role = admin` | Admin users only |

## Request Conventions

### Pagination

Offset-based pagination (consistent with current implementation):

```
GET /api/v1/products?page=2&limit=20
```

| Param | Type | Default | Max | Notes |
|-------|------|---------|-----|-------|
| `page` | integer | 1 | — | 1-indexed |
| `limit` | integer | 10 | 100 | Items per page |

### Sorting

```
GET /api/v1/products?sort=created_at&order=desc
GET /api/v1/orders?sort=total_amount&order=asc
```

| Param | Values | Default |
|-------|--------|---------|
| `sort` | column name (whitelisted per endpoint) | `created_at` |
| `order` | `asc` \| `desc` | `desc` |

### Filtering

```
GET /api/v1/products?category_id=5&is_active=true
GET /api/v1/products?min_price=100000&max_price=500000
GET /api/v1/orders?status=pending,confirmed
```

### File Upload

```
Content-Type: multipart/form-data

Field name:   file (single) | files (multiple)
Max size:     5 MB per file
Allowed MIME: image/jpeg, image/png, image/webp
```

## Response Format

All responses are wrapped in a standard envelope by `TransformInterceptor`.

### Success — Single Resource

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Áo thun nam",
    "slug": "ao-thun-nam"
  },
  "message": "Created successfully"
}
```

### Success — Paginated List

```json
{
  "success": true,
  "data": [ { "id": 1 }, { "id": 2 } ],
  "meta": {
    "page": 1,
    "limit": 10,
    "total": 95,
    "totalPages": 10
  }
}
```

### Error

```json
{
  "success": false,
  "error": {
    "code": "PROD_001",
    "message": "Product not found",
    "details": {}
  }
}
```

### Validation Error (field-level)

```json
{
  "success": false,
  "error": {
    "code": "SYS_002",
    "message": "Validation failed",
    "details": {
      "email": "must be a valid email address",
      "password": "must be at least 8 characters"
    }
  }
}
```

## Error Code Reference

Format: `{FEATURE}_{3-digit sequential number}`

| Code | HTTP | Description |
|------|------|-------------|
| **Auth** |||
| `AUTH_001` | 401 | Invalid credentials |
| `AUTH_002` | 401 | Access token expired |
| `AUTH_003` | 401 | Access token invalid |
| `AUTH_004` | 401 | Refresh token expired or revoked |
| `AUTH_005` | 403 | Insufficient permissions |
| `AUTH_006` | 409 | Email already registered |
| **User** |||
| `USER_001` | 404 | User not found |
| `USER_002` | 400 | Current password incorrect (change-password) |
| **Address** |||
| `ADDR_001` | 404 | Address not found |
| `ADDR_002` | 403 | Address does not belong to current user |
| **Product** |||
| `PROD_001` | 404 | Product not found |
| `PROD_002` | 404 | Product variant not found |
| `PROD_003` | 400 | Insufficient stock for variant |
| `PROD_004` | 404 | Category not found |
| **Cart** |||
| `CART_001` | 404 | Cart not found |
| `CART_002` | 400 | Cart is empty |
| `CART_003` | 404 | Cart item not found |
| **Order** |||
| `ORD_001` | 404 | Order not found |
| `ORD_002` | 403 | Order does not belong to current user |
| `ORD_003` | 400 | Cannot cancel — order status is `shipping` or `delivered` |
| **Review** |||
| `REV_001` | 409 | User already reviewed this product for this order |
| `REV_002` | 403 | Must purchase product before reviewing |
| `REV_003` | 404 | Review not found |
| **System** |||
| `SYS_001` | 500 | Internal server error — see server logs |
| `SYS_002` | 400 | Validation failed — see `details` for field errors |
| `SYS_003` | 429 | Rate limit exceeded |

### HTTP Status Mapping

| Status | When to use |
|--------|-------------|
| 200 | Successful read or update |
| 201 | Successful resource creation |
| 204 | Success with no response body (e.g., DELETE) |
| 400 | Client input error (validation, business rule) |
| 401 | Authentication required or failed |
| 403 | Authenticated but not authorized |
| 404 | Resource not found |
| 409 | Conflict (duplicate, state violation) |
| 429 | Rate limit exceeded |
| 500 | Unhandled server error |

## Endpoint Details

### POST `/auth/register`

**Request:**
```json
{
  "email": "user@example.com",
  "password": "securePassword123",
  "fullName": "Nguyen Van A",
  "phone": "0901234567"
}
```

**Validation:**
- `email`: valid email format, max 255 chars
- `password`: min 8 chars
- `fullName`: not empty, max 100 chars
- `phone`: optional, max 20 chars

**Response 201:**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "email": "user@example.com",
    "fullName": "Nguyen Van A",
    "role": "customer"
  },
  "message": "Registration successful"
}
```

| Error | Condition |
|-------|-----------|
| `AUTH_006` 409 | Email already registered |
| `SYS_002` 400 | Invalid email format, password too short |

---

### POST `/auth/login`

**Request:**
```json
{
  "email": "user@example.com",
  "password": "securePassword123"
}
```

**Response 200:**
```json
{
  "success": true,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "email": "user@example.com",
      "fullName": "Nguyen Van A",
      "role": "customer"
    }
  }
}
```

Response also sets:
```
Set-Cookie: refreshToken=<hashed_value>; HttpOnly; Secure; SameSite=Strict; Path=/api/v1/auth/refresh; Max-Age=604800
```

**Rate limit:** 5 attempts per 60 seconds per IP. Returns `SYS_003` 429 when exceeded.

| Error | Condition |
|-------|-----------|
| `AUTH_001` 401 | Email not found or password incorrect |

---

### POST `/orders/checkout`

**Request:**
```json
{
  "addressId": 5,
  "paymentMethod": "cod",
  "note": "Call before delivery"
}
```

**Validation:**
- `addressId`: integer, required — must belong to current user
- `paymentMethod`: one of `cod`, `bank_transfer`, `momo`
- `note`: optional string, max 500 chars

**Response 201:**
```json
{
  "success": true,
  "data": {
    "id": 100,
    "status": "pending",
    "paymentMethod": "cod",
    "paymentStatus": "unpaid",
    "shippingFee": 30000,
    "totalAmount": 530000,
    "shippingAddress": {
      "fullName": "Nguyen Van A",
      "phone": "0901234567",
      "addressLine": "123 ABC Street",
      "city": "Ho Chi Minh"
    },
    "items": [
      {
        "productName": "Áo thun nam",
        "sku": "ATN-WHITE-L",
        "price": 250000,
        "quantity": 2,
        "thumbnailUrl": "https://cdn.example.com/images/ao-thun-nam.jpg"
      }
    ],
    "createdAt": "2024-01-15T10:30:00.000Z"
  }
}
```

**Business logic executed in order:**
1. Verify cart is not empty → `CART_002` if empty
2. Verify `addressId` belongs to current user → `ADDR_001` / `ADDR_002`
3. Lock product variant rows (`SELECT FOR UPDATE`)
4. Validate stock for all items → `PROD_003` if any insufficient
5. `BEGIN TRANSACTION`
6. Deduct `stock_quantity` per variant
7. Snapshot shipping address from address record
8. Create `orders` row
9. Create `order_items` rows with snapshots (name, SKU, price, thumbnail)
10. Clear cart items
11. `COMMIT`
12. Emit `order.created` event (async: send confirmation email)

| Error | Condition |
|-------|-----------|
| `CART_002` 400 | Cart is empty |
| `ADDR_001` 404 | Address not found |
| `ADDR_002` 403 | Address belongs to different user |
| `PROD_003` 400 | Insufficient stock — includes which variant |

---

### POST `/products/:id/reviews`

**Request:**
```json
{
  "orderId": 100,
  "rating": 5,
  "comment": "Great product, fast delivery!"
}
```

**Validation:**
- `orderId`: integer, required — order must belong to current user and contain this product
- `rating`: integer 1–5, required
- `comment`: optional string, max 1000 chars

**Response 201:**
```json
{
  "success": true,
  "data": {
    "id": 50,
    "rating": 5,
    "comment": "Great product, fast delivery!",
    "createdAt": "2024-01-20T15:00:00.000Z",
    "user": {
      "id": 1,
      "fullName": "Nguyen Van A"
    }
  }
}
```

**Business logic:**
1. Verify order belongs to current user
2. Verify product appears in the order's `order_items`
3. Verify no existing review for this `(user_id, product_id, order_id)` combination

| Error | Condition |
|-------|-----------|
| `ORD_001` 404 | Order not found |
| `ORD_002` 403 | Order belongs to different user |
| `REV_002` 403 | Product not in this order (purchase not verified) |
| `REV_001` 409 | Review already exists for this order + product |

## NestJS Implementation Notes

```typescript
// Controller pattern for protected endpoint
@ApiTags('orders')
@ApiBearerAuth()
@Controller('orders')
@UseGuards(JwtAuthGuard)
export class OrderController {
  constructor(private readonly checkoutService: CheckoutService) {}

  @Post('checkout')
  @HttpCode(HttpStatus.CREATED)
  @ApiOperation({ summary: 'Create order from cart' })
  @ApiResponse({ status: 201, type: OrderResponseDto })
  async checkout(
    @CurrentUser() user: IUserPayload,
    @Body() dto: CreateOrderDto,
  ): Promise<OrderResponseDto> {
    return this.checkoutService.execute(user.id, dto);
  }
}

// Rate limiting on auth endpoints
@Throttle({ default: { limit: 5, ttl: 60_000 } })
@Post('login')
async login(@Body() dto: LoginDto): Promise<LoginResponseDto> {
  return this.authService.login(dto);
}
```
